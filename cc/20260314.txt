10:Te9b,<h2>为什么同一个命令会有多个安装路径？</h2>
<p>在 macOS 上，同一个工具（如 <code>claude</code>、<code>42plugin</code>）可能通过不同方式安装多次：</p>
<ul>
<li><code>brew install</code> → <code>/opt/homebrew/bin/</code></li>
<li><code>npm/bun install -g</code> → <code>~/.bun/bin/</code> 或 <code>/opt/homebrew/lib/node_modules/</code></li>
<li>官方安装器 → <code>~/.local/bin/</code></li>
</ul>
<p>用 <code>where</code> 命令可以看到所有安装路径：</p>
<pre><code class="language-bash">where claude
# /Users/admin/.local/bin/claude
# /Users/admin/.bun/bin/claude
# /opt/homebrew/bin/claude
</code></pre>
<hr>
<h2>系统实际执行哪一个？</h2>
<p>取决于 <code>PATH</code> 环境变量的顺序。<code>PATH</code> 从左到右查找，<strong>找到第一个就停止</strong>。</p>
<pre><code class="language-bash">echo $PATH
# /Users/admin/.local/bin:/Users/admin/.bun/bin:/opt/homebrew/bin:...
</code></pre>
<p><code>where</code> 输出的<strong>第一行</strong>就是实际执行的版本。用 <code>which</code> 可以直接确认：</p>
<pre><code class="language-bash">which claude
</code></pre>
<h3>为什么不同用户顺序不同？</h3>
<p><code>PATH</code> 是在 <code>~/.zshrc</code> 里叠加的，<strong>越晚 <code>export</code> 的越靠前</strong>：</p>
<pre><code class="language-bash">export PATH="$HOME/.bun/bin:$PATH"    # bun 排前
export PATH="$HOME/.local/bin:$PATH"  # local 插队到最前（最终最靠前）
</code></pre>
<hr>
<h2>如何判断哪个版本该保留？</h2>
<p>用 <code>file</code> 命令查看文件类型：</p>
<pre><code class="language-bash">file ~/.local/bin/claude ~/.bun/bin/claude /opt/homebrew/bin/claude
</code></pre>
<p>输出示例：</p>
<pre><code>~/.local/bin/claude    Mach-O 64-bit executable arm64   ← 原生二进制
~/.bun/bin/claude      a /usr/bin/env node script       ← Node.js 脚本
/opt/homebrew/bin/claude  a /usr/bin/env node script    ← Node.js 脚本
</code></pre>
<table>
<thead>
<tr>
<th>类型</th>
<th>说明</th>
<th>优先级</th>
</tr>
</thead>
<tbody>
<tr>
<td>Mach-O 原生二进制</td>
<td>官方安装器分发，arm64 原生，启动快，不依赖 Node.js</td>
<td>✅ 保留</td>
</tr>
<tr>
<td>Node.js 脚本</td>
<td>npm/bun/brew 安装的旧格式</td>
<td>❌ 清理</td>
</tr>
</tbody>
</table>
<p>如果有多个原生二进制，比较版本号和修改时间：</p>
<pre><code class="language-bash">ls -la ~/.local/bin/42plugin /opt/homebrew/bin/42plugin
~/.local/bin/42plugin --version
/opt/homebrew/bin/42plugin --version
</code></pre>
<p>保留版本号更高、时间更新的。</p>
<hr>
<h2>清理多余安装</h2>
<h3>情况一：brew 安装的</h3>
<pre><code class="language-bash">brew uninstall &#x3C;package-name>
</code></pre>
<h3>情况二：npm 全局安装的（symlink 在 <code>/opt/homebrew/bin/</code>）</h3>
<pre><code class="language-bash"># 查看 symlink 指向
ls -la /opt/homebrew/bin/claude
# → ../lib/node_modules/@anthropic-ai/claude-code/cli.js

# 用 npm 卸载（更干净）
npm uninstall -g @anthropic-ai/claude-code

# 或直接删除
rm /opt/homebrew/bin/claude
rm -rf /opt/homebrew/lib/node_modules/@anthropic-ai/claude-code
</code></pre>
<h3>情况三：bun 全局安装的</h3>
<pre><code class="language-bash">bun remove -g &#x3C;package-name>
# 或直接
rm ~/.bun/bin/&#x3C;binary-name>
</code></pre>
<hr>
<h2>总结流程</h2>
<pre><code>where &#x3C;cmd>          # 找出所有安装路径
file &#x3C;路径1> &#x3C;路径2>  # 判断类型（原生 vs Node.js）
which &#x3C;cmd>          # 确认当前生效的版本
                     # 删除多余版本，保留原生二进制
</code></pre>
